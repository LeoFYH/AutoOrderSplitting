# 学校订单自动拆单工具

把苏东坡导出的订单明细按采购模板里的商品规则拆成采购单。

## 当前实现

- 读取采购模板，提取 `商品名称 -> 供应商 / 采购单价 / 模板备注`。
- 读取一个或多个订单导出文件，跳过 `小计:` 汇总行。
- 商品先精确匹配，再做归一化匹配，最后做确定性的模糊匹配。
- 模板匹配优先；模板找不到学校/商品时，如果订单里有 `默认供应商`，使用订单自带供应商和订单单价兜底。
- 按 `供应商 + 学校 + 商品` 聚合数量。
- 输出 `采购单.xlsx`。
- 页面里直接显示未匹配、跳过和警告。

## 本地 Web 运行

在 PowerShell 里进入项目目录后运行：

```powershell
.\start_web.ps1
```

页面会打开：

```text
http://127.0.0.1:5000
```

使用顺序：

1. 上传采购模板。
2. 在页面里按需修改模板行并保存。
3. 批量选择订单文件。
4. 一键生成并下载 `采购单.xlsx`。

## 命令行运行

### 安装依赖

```powershell
python -m pip install -e .
```

### 运行

```powershell
python -m auto_order_splitting `
  --template "C:\Users\11752\Downloads\采购模板.xlsx" `
  --orders "C:\Users\11752\Downloads\Copy of 订单导出1780712548.xlsx" `
  --output ".\outputs\采购单.xlsx"
```

默认会把模板已有的数据行也带入并聚合，适合模板里已经有当天固定采购项的场景。
如果只想按订单生成，不带模板里的数量行，加：

```powershell
--orders-only
```

默认跳过订单学校字段命中 `营养餐` 的订单行；也可以在 Web 页面里点 `+ 加关键词` 配置多项，例如 `食材配送`、`早晚餐`。学校不在模板中会进入未匹配，不会被当成跳过项丢掉。如果不想默认跳过，加：

```powershell
--no-default-skip
```

默认即使存在未匹配行也会生成结果和报告并正常结束。需要在自动化脚本里把未匹配视为失败时，加：

```powershell
--fail-on-unmatched
```

## 输出文件

- `采购单.xlsx`：成品采购单。
- 未匹配、跳过和警告直接在 Web 页面底部查看。
